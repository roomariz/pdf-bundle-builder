# Release Checklist (Windows)

This project ships Windows executables via GitHub Actions + PyInstaller.

## Before Tagging

1. Run tests locally:

```bash
uv sync --extra dev
uv run pytest
```

1b. Update GitHub sidebar metadata (manual):

- Description:
  - Privacy-first offline PDF evidence bundle builder with TOC, dividers, page numbers, CLI and desktop app.
- Topics:
  - pdf, python, desktop-app, customtkinter, reportlab, legal-tech, privacy, offline, productivity, immigration, evidence, pdf-merger

2. Sanity-check the CLI and GUI locally:

```bash
uv run pdf-bundle-builder merge --input sample-input --output /tmp/out.pdf
uv run python -m pdf_merger.app
```

3. Confirm the repo contains only safe demo PDFs:

- `sample-input/` should contain only dummy documents generated by `scripts/generate_sample_pdfs.py`
- No personal/private PDFs should exist in git history

## Create the Release Tag

1. Pick a version tag: `vX.Y.Z` (example: `v0.2.0`)
2. Tag and push:

```bash
git tag vX.Y.Z
git push origin vX.Y.Z
```

## Verify GitHub Actions Output

1. Open GitHub Actions and ensure `release-windows` ran for the tag.
2. Confirm the workflow produced:

- `pdf-bundle-builder.exe` (CLI)
- `pdf-bundle-builder-gui.exe` (GUI, no console window)
- `pdf-bundle-builder-windows.zip` containing both exes + `README.md` + `LICENSE`

## Smoke Test the Release Zip (Windows)

1. Download `pdf-bundle-builder-windows.zip` from the GitHub Release.
2. Extract it and run:

```powershell
.\pdf-bundle-builder.exe --help
.\pdf-bundle-builder.exe merge --input .\sample-input --output .\out.pdf
.\pdf-bundle-builder-gui.exe
```

3. Verify:

- merged PDF is created
- TOC shows final page numbers
- GUI starts without a console window
